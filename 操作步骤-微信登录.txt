═══════════════════════════════════════════════════════════
  如何让「微信登录」不 404（PC + 手机都能用）
═══════════════════════════════════════════════════════════

【重要】必须用「带后端的启动方式」，不能用「启动本地预览.bat」。

───────────────────────────────────────────────────────────
一、PC 端操作
───────────────────────────────────────────────────────────

1. 双击运行：  启动带登录.bat

2. 等待黑窗口里出现类似：
     Server: http://localhost:3000
     (未配置微信 AppID，当前为模拟登录)

3. 用浏览器打开：  http://localhost:3000

4. 再点页面上的「微信登录」
   → 会直接模拟登录成功，不再 404。


───────────────────────────────────────────────────────────
二、手机端操作（和电脑同一 WiFi）
───────────────────────────────────────────────────────────

1. 电脑上先运行：  启动带登录.bat  （保持黑窗口不关）

2. 查电脑本机 IP：
   - 按 Win + R，输入 cmd 回车
   - 输入：  ipconfig
   - 找到「无线局域网适配器」或「以太网」里的
     IPv4 地址，例如：  192.168.1.100

3. 手机浏览器地址栏输入：
     http://192.168.1.100:3000
   （把 192.168.1.100 换成你电脑上看到的 IP）

4. 再在手机页面上点「微信登录」
   → 同样会走模拟登录，不 404。


───────────────────────────────────────────────────────────
三、若仍然 404，请检查
───────────────────────────────────────────────────────────

□ 是否运行的是「启动带登录.bat」，而不是「启动本地预览.bat」？
□ 浏览器地址是不是 http://localhost:3000（PC）或 http://电脑IP:3000（手机）？
□ 黑窗口是否一直开着、没有报错？
□ 若第一次运行，是否等「npm install」装完再打开浏览器？


───────────────────────────────────────────────────────────
四、若是部署到网上（如 GitHub Pages）
───────────────────────────────────────────────────────────

GitHub Pages 只有静态页面，没有 /api 接口，所以点「微信登录」一定会 404。

要支持登录，需要：
- 把 server 文件夹里的后端部署到能跑 Node 的服务器（如云主机、Railway、Render 等），
- 在前端把 API 地址改成该后端的网址。

本地按上面一、二步操作即可避免 404。
